## 趨勢分析

### 2026-06-14（本次同步與 UI 重構）
- 本次對話新增了 5 個新教訓（教訓 #8 ~ #12），其中 #8 與 #9 為上一階段，#10 ~ #12 為本階段新發現：
  - **#10 (環境 - TW-04)**: 全新 Git Worktree 忽略了依賴缺失，直接跑建置失敗，加上連線不穩。浪費 ~9,000 tokens。
  - **#11 (前端 - 無)**: 系統設定頁面關鍵字清單 map 迴圈使用重複 key 導致的渲染衝突。0 tokens。
  - **#12 (前端 - TW-01)**: Next.js 的 suppressHydrationWarning 僅限單層生效，導致漏掉 body 標籤。浪費 ~3,000 tokens。
- **核心教訓：全新 Worktree 必先還原依賴；React 清單 key 必須保證唯一；suppressHydrationWarning 只防單層。**

### 2026-04-22（歷史首次記錄）
- 發現 4 個教訓，最嚴重問題為 Angular SPA 頁面動態 Selector 失敗（TW-07 + TW-02），最終以 `inner_text` 解決，浪費 ~30,000 tokens。

## 改善目標

1. **TW-07 清零**: 未來遇到 SPA 頁面，第一時間用文字解析，不嘗試 selector。
2. **TW-04 清零**: 掃描目錄前確認分層深度，在 Windows 上執行 Python 前先設 `PYTHONIOENCODING`。
3. **TW-02 清零**: 修改 JSON 欄位結構時，一併審查並重構 React 前端渲染邏輯，本地 build 測試通過後再 push。
