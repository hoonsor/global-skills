## 趨勢分析

### 2026-06-14（本次同步與 UI 重構）
- 本次對話發現 2 個新教訓，涉及 `TW-04`（冷凍目錄層級假設錯誤）與 `TW-02`（僅同步數據漏改前端 UI）。
- 最嚴重問題為前端未同步更新即推送部署，導致網頁在同步數據後仍無顯示，經過 Next.js 本地 Build 測試與前端代碼修改後修復。
- **核心教訓：新資料欄位必須同步修改 UI 組件；複雜掃描務必核對目錄分層深度。**

### 2026-04-22（歷史首次記錄）
- 發現 4 個教訓，最嚴重問題為 Angular SPA 頁面動態 Selector 失敗（TW-07 + TW-02），最終以 `inner_text` 解決，浪費 ~30,000 tokens。

---

## 改善目標

1. **TW-07 清零**: 未來遇到 SPA 頁面，第一時間用文字解析，不嘗試 selector。
2. **TW-04 清零**: 掃描目錄前確認分層深度，在 Windows 上執行 Python 前先設 `PYTHONIOENCODING`。
3. **TW-02 清零**: 修改 JSON 欄位結構時，一併審查並重構 React 前端渲染邏輯，本地 build 測試通過後再 push。
