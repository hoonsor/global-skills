## 趨勢分析

### 2026-04-22（首次記錄）
- 本次對話發現 4 個教訓，3 種浪費模式
- 最嚴重問題：**Angular SPA 頁面嘗試用 CSS Selector 抓取動態內容**（TW-07 + TW-02）
  - 總共嘗試 10+ 輪，每輪修改 selector / JS 注入策略
  - 最終解法極為簡單：`page.inner_text('body')` 一行解決
  - **核心教訓：簡單方法往往是最好的方法（TW-05 隱性觸發）**

---

## 改善目標

1. **TW-07 清零**: 未來遇到 SPA 頁面，第一時間用文字解析，不嘗試 selector
2. **TW-04 清零**: 在 Windows 上執行 Python 前，永遠先設定 `PYTHONIOENCODING`
3. **TW-06 減半**: pip 安裝後立即用 `where` 和 `pip show` 驗證
